# AI Agent 开发规则

## 核心原则

- **效率至上**：快速单元式开发
- **不写文档**：只写代码，不创建 README、GUIDE 等文档文件
- **改完即退**：完成代码修改后立即退出，用户会手动测试
- **单元提交**：每个功能/修改单独提交到 Git
- **闭嘴**：非用户要求不输出任何内容，静默更改代码完毕后直接退出
- **中文**：和开发者交流都必须是中文，不能是英文或其他语言

## 工作流程

1. 理解需求
2. 编写/修改代码
3. 创建 Git 提交
4. 退出（不等待测试结果）

## 提交规范

- `feat: ` - 新增业务功能
- `fix: ` - 修复程序 Bug、逻辑异常
- `refactor: ` - 代码重构，无新增功能、无 Bug 修复
- `style: ` - 代码格式、缩进、标点调整，不影响业务逻辑
- `perf: ` - 性能优化、加载提速、执行效率优化
- `chore: ` - 构建配置、工程工具、依赖及杂项更新
- `test: ` - 新增 / 修改单元测试、测试用例
- `docs: ` - 文档、注释、说明文案变更
- `build: ` - 构建系统、打包配置、外部依赖变更
- `ci: ` - CI 流水线、GitHub Actions、Docker 配置变更
- `revert: ` - 回退历史 Commit 提交
- friend-link: ` - 新增好友链接
- friend-link: ` - 更新好友链接
